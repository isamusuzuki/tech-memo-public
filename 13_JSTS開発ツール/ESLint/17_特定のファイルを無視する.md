# 特定のファイルを無視する

作成日 2025/10/01、更新日 2025/11/05

## 1. 公式ドキュメントを読む（英語）

[Ignore Files](https://eslint.org/docs/latest/use/configure/ignore#ignoring-files)

`eslint.config.js`ファイルの中で、`globalIgnores()`ヘルパー機能を使い、無視するファイルのパターンを指示することができる

```javascript
import { defineConfig, globalIgnores } from "eslint/config";

export default defineConfig([
    globalIgnores([".config/*"])
]);
```

## 2. `--no-warn-ignored`オプション

huskyとlint-stagedを使って、コミットする際に必ずESLintが走るように設定している

pacakge.json

```json
{
    "lint-staged": {
        "src/**/*.ts": "eslint --max-warnings 0 --no-warn-ignored"

    }
}
```

- `--max-warnings 0`オプションで、1個でもエラー・警告があればコミットが失敗するようになっている
- では、`--no-warn-ignored`オプションは何をしているのか？

### 公式リファレンス（英語）を読む

[ESLint CLI Reference > --no-warn-ignored](https://eslint.org/docs/latest/use/command-line-interface#--no-warn-ignored)

ESLint でファイルをチェックするとき、無視される（ignored）ファイルを指定すると、普通は次のような警告が出る

```text
File ignored because of a matching ignore pattern.
File ignored by default.
```

`--no-warn-ignored`を使うことで、無視されるファイルを指定しても、この警告が出なくなる

`--max-warnings 0`は、「警告が1つでも出たらエラー扱いにする（終了コード1を返す）」という設定で、ignored ファイルに対する警告も「警告」として数えられるため、意図せず ESLint が失敗してしまうことがある

だから、この2つを併用することで、「無視ファイルに関する警告を出さず、純粋な警告ゼロチェック」ができるようになる
